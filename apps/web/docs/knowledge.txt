5) LEAD INTAKE WORKFLOW (CANONICAL)
==================================================
Name: Demo Request → Stage Engine → Task Queue

Input channels:
- Website “Request a Demo” form (preferred)
- Email forwarding alias (fallback)
- Manual entry (admin)

Canonical lead object fields:
- first_name, last_name
- email, phone
- company, role, industry
- what_to_automate (free text)
- source (Website Form / Referral / Cold Outreach / Other)
- status (New Lead / Contacted / Discovery Scheduled / Proposal Sent / Closed Won / Closed Lost)
- created_at, last_contact_at, last_submission_at
- meeting_link

Core rules:
- If email exists: update last_submission_at and merge new details
- If new: create record + notify owner
- After status changes: generate task templates (follow-up, discovery prep, proposal prep)

6) STAGE DEFINITIONS (PIPELINE)
==================================================
Stage: New Lead
Entry criteria:
- A lead record exists with status = New Lead
Exit criteria:
- Status updated to Contacted or Discovery Scheduled
Automation notes:
- Send internal notification + schedule a follow-up task if no response in 6 hours

Stage: Contacted
Entry criteria:
- First outreach sent
Exit criteria:
- Meeting booked OR marked unresponsive
Automation notes:
- Re-send booking link after 48 hours if no booking

Stage: Discovery Scheduled
Entry criteria:
- Meeting link exists + calendar event detected
Exit criteria:
- Discovery complete OR no-show
Automation notes:
- AI summarizes lead details into discovery brief

Stage: Proposal Sent
Entry criteria:
- Proposal delivered
Exit criteria:
- Closed Won/Lost
Automation notes:
- Reminders at 3 and 7 days; auto-create “proposal follow-up” tasks

Stage: Closed Won
Entry criteria:
- Agreement signed OR invoice paid
Exit criteria:
- (n/a)
Automation notes:
- Kick off onboarding checklist + welcome email

Stage: Closed Lost
Entry criteria:
- Explicit decline OR no response after 21 days
Automation notes:
- Tag reason if known; archive and log notes

- trigger_stage: Discovery Scheduled
- offset: 2 Hours before meeting
- required: true

- type: Proposal Prep
- trigger_stage: Discovery Scheduled
- offset: 1 Day after meeting
- required: true
